Task Definition

Task: numerical reasoning QA over Vietnamese financial documents (text + table) — generate an executable computation program, not just the final number, so every prediction is auditable.

The program — not just the final number — is the model's output: a transparent, verifiable reasoning path built from a fixed set of arithmetic and table operators.

Datasets & Metrics

  • FinQA (Chen et al., 2021) — English financial reports, text + table → program
  • ViNumQA (Toan et al. 2025) — Vietnamese financial reports, same task format
Table 1: Train / valid / test split sizes for FinQA and ViNumQA
Dataset Train Valid Test Total
FinQA 6,251 883 1,147 8,281
ViNumQA 2,993 584 497 4,074

Metrics:

  1. Program Accuracy (PA) primary metric — exact structural match to the gold program. Prioritized because it shows transparency: a numerically correct answer from a logically wrong program is considered meaningless in this domain.
  2. Execution Accuracy (EA) reference only — executing the program yields the correct answer

FinQA: A Dataset of Numerical Reasoning over Financial Data (Chen et al., EMNLP 2021) (A* conference)

VLSP 2025 challenge: Numerical Reasoning Question and Answer (Toan et al., VLSP 2025)

Approach & Roadmap

A progressively stronger set of approaches on the same held-out set — click a stage to expand.

1 In-context learning
Setup: 0/1/few-shot prompting
Models: Qwen3-4B, Llama-3.3-70B, DeepSeek-V4-Flash, gpt-5-nano — no fine-tuning, no training data needed; establishes the zero/low-resource baseline
2 Reasoning trace distillation
Setup: teacher → student
Teacher: Qwen3-Next-80B-A3B-Thinking rationalizes gold programs into <think>...</think> reasoning traces (CoNR)
3 Supervised fine-tuning
Setup: QLoRA, w/o & w/ reasoning trace
Data: ViNumQA (pre_text + table + question → program), with and without the distilled reasoning traces
Model: Qwen3-4B with PEFT and 4-bit quantization
4 SFT + RL planned
Setup: GRPO
Plan: reward on program validity + execution correctness + conciseness, starting from the SFT checkpoint to push past the SFT ceiling
5 Multi-agent planned
Plan: planner / executor / verifier roles to catch gold-label errors and self-correct programs before final answer
